GPG Keychain Access

Developer's Roadmap

05 January 2005

This document provides a development scheme for GPG Keychain Access. This scheme is rough, as it does not anticipate any external changes to the computing environment in which it will be developed or used. Changes in that computing environment will neccesitate changes to this roadmap.

With the release of version 0.7.0, development of GPG Keychain Access will split into two branches. The first branch, 0.7, will be a maintenance branch that maintains compatibility with Mac OS X 10.2 (Jaguar), while the second branch, 0.8 and its decendents will be compatable with Mac OS X 10.3 (Panther) and will contain new features.

0.6.1: Current Version

0.7: January 2005

  • GUI to control key generation
  • Online help
  • Partial Address Book support

0.8: Summer 2005

  • Full Address Book support
  • iCal support
  • GUI for some key editing
  • Migrate all key manipulation functions from Terminal.app to GPGME or pipes
  • New GUI for key importing and exporting
  • GPG->Import Key system service

Much of the application will be refactored to make future maintainance easier and quicker, particularly the updating of translations. In the fall of 2004, GPGME 1.0 was released and this verison of GPGME will be incorporated into GPG Keychain Access and other products of the MacGPG project, leading to significantly improved performance. Work on GPG Keychain Access 0.8 will wait on the production of GPGME.framework 1.0.

1.0: Autumn 2005

  • Additional key editing using GUI mechanisms

1.1: Winter or Spring 2006

  • Add a key signing party helper of some sort?
